Incoherences entre composants

1. Marquage champs requis

Certains formulaires utilisent *, d'autres s'appuient uniquement sur l'attribut HTML required. Pas de pattern visuel consistant.

2. Affichage erreurs

Auth : alert boxes. Modals creation : toast uniquement. Settings : toast. Pas de messages d'erreur au niveau champ nulle part.

3. Trim des champs texte

Creer tiers : trim ajoute. Register : pas de trim. Creer batiment : pas de trim. Inconsistant.

4. Validation email

HTML5 type="email" partout mais pas de regex validation. Accepte des formats invalides.

5. Validation telephone

Aucune validation de format nulle part. Accepte n'importe quel texte.

6. Confirmation actions destructives

Annulation mission : confirmation. Unlink proprio : pas de confirmation. Suppression adresse : pas de confirmation. Delete indispo : confirmation. Inconsistant.

7. Loading states boutons

Login : texte change + disabled. Creer mission : disabled seul. Creer tiers : pas de loading visible. Devrait etre consistant.

8. Empty state icone

Patrimoine : rounded-2xl. Tiers : rounded-full. Missions : rounded-full. Devrait etre un seul pattern.

9. Edit mode pages detail

Toutes les pages utilisent FloatingSaveBar maintenant — BIEN. Mais mission detail permet encore de modifier des champs en mode terminee cote UI.

10. Data refresh apres mutations

Building : pas d'invalidation apres address mutation. Mission : pas d'invalidation apres assign tech / update cle. Tiers : pas d'invalidation orga link.

11. Section collapse non persistee

Mission/tiers detail : sections collapsibles remises a defaut au reload. Devrait sauver dans localStorage.

12. Responsive breakpoints

Building detail : grid-cols-2 sans breakpoint mobile. Patrimoine table : colonnes overflow sur mobile.

5 Bugs critiques

1. Mission terminee editable cote UI

mission-detail-page.tsx — Les champs planning (date, heure, statut_rdv) sont modifiables meme quand isTerminated=true. Le backend rejette mais l'UI ne bloque pas. L'utilisateur croit pouvoir modifier.

Fix: Desactiver les inputs quand isLocked=true (disabled prop)

2. Form reset sur refetch mission

mission-detail-page.tsx:89 — useEffect([mission, editing]) sync le form a chaque changement de mission. Si l'API refetch pendant que l'user edite, ses modifs sont ecrasees sans warning.

Fix: Ajouter un dirty flag, ne pas sync si le form a ete modifie

3. Import CSV sans rollback

import-csv-modal.tsx — Si la ligne 5 echoue, les lignes 1-4 sont deja commitees en base. Pas de transaction. Peut creer des batiments orphelins.

Fix: Utiliser une transaction backend ou un batch insert atomique

4. Debounce hook patrimoine bugge

patrimoine-page.tsx:87 — useDebounce utilise useState de facon incorrecte (mutation synchrone dans le render). Peut causer des race conditions sur la recherche.

Fix: Reimplementer avec useEffect + setTimeout classique

5. Indisponibilite recurrence count=0

unavailability-modal.tsx — Le champ count accepte 0 ou negatif. Cree une recurrence infinie ou invalide cote backend.

Fix: Ajouter min="1" et validation count > 0 avant submit
