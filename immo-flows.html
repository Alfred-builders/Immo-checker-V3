Admin / Gestionnaire
Créer la mission (modal)
Depuis dashboard, page Missions, ou fiche bâtiment
  • Lot (picker enrichi : nom + adresse + propriétaire + étage)
  • Sens : Entrée / Sortie / Entrée+Sortie simultanées
  • Avec inventaire (toggle)
  • Locataire(s) + Type de bail si 2+ locataires (Individuel / Collectif)
  • Date, heure début/fin
  • Technicien (dropdown, 1 en V1)
  • Commentaire (instructions terrain)
  • Référence auto-générée : M-2026-XXXX
Système
Création automatique des EDL_Inventaire
Selon le sens + bail individuel (N EDL) ou collectif (1 EDL + N locataires via pivot)
  • Bail individuel → 1 EDL par locataire
  • Bail collectif → 1 EDL, N locataires dans EDLLocataire
  • Entrée+Sortie → EDL.sens distingue entrant/sortant
Système
Envoi invitation au technicien
MissionTechnicien.statut_invitation = en_attente. Email Resend envoyé.
  • statut_rdv = a_confirmer (en attente confirmation locataire)
  • Mission apparaît dans "Actions en attente" du dashboard admin
Technicien
Accepter ou Refuser la mission
Sur app mobile ou bureau
  • Accepte → statut_invitation = accepte → Mission passe en "assignee"
  • Refuse → statut_invitation = refuse → Mission retourne dans "Actions en attente"
Admin / Gestionnaire
Gérer les indisponibilités technicien (optionnel)
Créneaux horaires, journée entière, récurrence Google Calendar (quotidien/hebdo/bimensuel/mensuel)
  • Affiché en plages grisées dans le calendrier admin
  • CRUD depuis dashboard (bouton "+") ou app mobile technicien
Technicien (terrain — App tablette)
Réaliser l'EDL sur tablette
EPIC 5 — hors périmètre webapp. Saisie pièces, items, photos, relevés compteurs.
  • Code accès saisi sur EDL_Inventaire (pas sur le lot)
  • Colocation : peut dupliquer l'EDL (changer locataire + ajuster pièces)
  • Documenter les clés : type, quantité, statut
Locataire
Signer l'EDL
Signature sur tablette. EDL.statut passe de brouillon → signe (document légal irrévocable).
  • Ou statut → infructueux si EDL ne peut être réalisé
  • QR code de vérification d'authenticité généré
Système (auto-terminaison)
Mission → terminee automatiquement
Quand TOUS les EDL liés à la mission sont signés. Colocation = attend le dernier signataire.
  • Date/heure/technicien/statut RDV passent en lecture seule
  • PDF et liens web générés pour chaque EDL signé
  • Clés : statut "a_deposer" → "deposee" reste modifiable
Admin / Gestionnaire
Consulter la mission terminée
Fiche mission : EDL signés, liens PDF/Web, timeline historique. Gestion dépôt clés.
  • 4 liens par EDL signé : PDF, Web, PDF légal, Web légal
  • Statut dépôt clés : remise / a_deposer / deposee
  • Commentaire toujours modifiable

Workflow Colocation (2+ locataires)
Bail individuel (défaut)
  • N locataires → N EDL distincts
  • Chaque EDL lié à 1 locataire
  • Technicien : peut dupliquer EDL sur tablette

  • Auto-terminaison attend TOUS signés
  • N clés (CleMission.edl_id) séparées
Bail collectif (solidaire)
  • N locataires → 1 seul EDL
  • N locataires liés via EDLLocataire (pivot)
  • Tous signent le même document
  • 1 seule CleMission
  • Auto-terminaison dès cet EDL signé
Entrée + Sortie simultanées Une mission peut contenir à la fois des EDL de sortie (ancien locataire) et d'entrée (nouveau locataire) sur le même lot. EDL.sens distingue les deux.

Connexions transversales clés

Super Admin → Workspace

  • Crée workspace (form + email admin)
  • Suspend/réactive workspace
  • Promeut/rétrograde super-admins
  • Renvoie invitations expirées
  • Lit l'audit log de toutes actions

Admin → Workspace

  • Configure workspace (nom, logo, SIRET...)
  • Invite utilisateurs (email + rôle)
  • Modifie rôles via dropdown inline
  • Désactive/supprime utilisateurs
  • Configure templates, catalogue, critères

Admin/Gestionnaire → Patrimoine

  • CRUD Bâtiment avec adresses multiples
  • CRUD Lot rattaché à bâtiment
  • Lie propriétaires via LotProprietaire
  • Lie mandataire via Lot.mandataire_id
  • Accès lots via drill-down ou recherche

Admin/Gestionnaire → Tiers

  • CRUD Tiers (physique ou morale)
  • Lie personne physique à organisation
  • Affecte rôles via relations (pas attribut)
  • Historique complet tous lots d'un locataire
  • Contacts d'une personne morale

Admin/Gestionnaire → Mission

  • Crée mission (lot immuable après création)
  • Assigne technicien (dropdown 1 seul)
  • Modifie date/heure/commentaire
  • Gère statut RDV (a_confirmer/confirme/reporte)
  • Annule si non terminée (motif obligatoire)

Technicien → Mission/EDL

  • Accepte/refuse invitation mission
  • Déclare indisponibilités
  • Réalise EDL sur tablette (EPIC 5)
  • Fait signer le locataire (déclenche terminaison)
  • Gère statut clés (a_deposer / deposee)
Périmètre webapp vs tablette La webapp (back-office) descend jusqu'au niveau Mission / EDL_Inventaire. La saisie terrain (pièces, items, photos, relevés compteurs) est exclusivement réalisée sur l'app tablette (EPIC 5). Le back-office consulte les données terrain uniquement via les PDF/liens Web générés après signature.
